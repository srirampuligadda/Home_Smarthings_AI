## ADDED Requirements

### Requirement: Routine Context Injection
The gemini-orchestrator SHALL inject the current state of devices, available SmartThings Routines, and recent sensor history logs into the Gemini prompt context.

#### Scenario: Generate system prompt with routine options
- **WHEN** a user initiates a conversation or requests an action
- **THEN** the orchestrator compiles the list of active Routines, device states, and sensor logs, and injects it as context for the Gemini model

### Requirement: Routine-Only Function Calling Scope
The gemini-orchestrator SHALL define and limit its function-calling tools exclusively to listing available Routines and staging routine trigger actions.

#### Scenario: Map intent to routine trigger tool call
- **WHEN** the user requests a complex command (e.g., "activate the movies setup")
- **THEN** the model is forbidden from calling individual device status tools and MUST call the routine trigger function matching the movie setup Routine ID

### Requirement: Action Staging Interception
The gemini-orchestrator SHALL intercept tool execution calls and stage them as pending actions rather than executing them directly.

#### Scenario: Intercept routine execution tool call
- **WHEN** Gemini outputs a routine trigger tool call
- **THEN** the orchestrator saves the tool payload with a pending state, appends it to `/api/pending-actions`, and notifies the user that the action is awaiting approval rather than executing immediately

### Requirement: Smart Home Strategy Consultant Role Prompting
The gemini-orchestrator SHALL enforce the AI model's role as a "Smart Home Strategy Consultant" that analyzes read-only sensor logs to propose optimization recommendations.

#### Scenario: Consultant recommends a new routine based on logs
- **WHEN** the user asks for suggestions or when sensor logs reveal optimization patterns
- **THEN** the orchestrator directs the Gemini model to respond using the exact "SmartThings Routine Blueprint" markdown structure detailing Why, If (Triggers), and Then (Actions)

### Requirement: Mandatory Routine Blueprint Template Formatting
The gemini-orchestrator SHALL enforce that all Routine optimization recommendations generated by the model match the specified Markdown template exactly.

#### Scenario: Verify template format output
- **WHEN** the consultant outputs a Routine recommendation
- **THEN** the output starts with `### 📋 Suggested New Routine: ` and contains the bullet points `* **Why:** `, `* **If (Triggers & Conditions):** `, and `* **Then (Actions):** ` in that order
